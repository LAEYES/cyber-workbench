# Compliance Checklist (GDPR / NIS2 / DORA / PCI DSS) — D01

**Organization:** ACME  
**Version:** 0.2 (hardened)  
**Date:** 2026-02-03

> **License-safe** checklist: summarized requirements (no copyrighted standard text). Adapt to status/scope.

## GDPR (minimum)
- [ ] Record of processing activities up to date (owners, purposes, subprocessors) — **evidence:** RoPA export
- [ ] Lawful basis documented for each processing — **evidence:** processing sheet
- [ ] DPIAs for high-risk processing — **evidence:** DPIA + approval
- [ ] Data breach procedure (≤ 72h notification) tested — **evidence:** procedure + exercise

## NIS2 (minimum)
- [ ] Security governance (roles, reporting) — **evidence:** ISP + RACI
- [ ] Formal risk management — **evidence:** risk register + reviews
- [ ] Incident handling + notification — **evidence:** playbooks + tickets
- [ ] Supply chain security — **evidence:** TPRM + clauses + vendor inventory

## DORA (minimum, if applicable)
- [ ] ICT risk management framework — **evidence:** policies + standards
- [ ] Resilience testing (at least yearly for critical scope) — **evidence:** test reports
- [ ] Critical ICT third-party management — **evidence:** vendor classification + audits
- [ ] Major incident reporting — **evidence:** procedure + RACI

## PCI DSS (minimum, if applicable)
- [ ] CDE scope identified and isolated — **evidence:** diagrams + network rules
- [ ] Vulnerability/patch management for CDE — **evidence:** SLAs + reports
- [ ] Centralized logging/audit trails — **evidence:** SIEM + retention
- [ ] Regular testing (scans/pentests per scope) — **evidence:** reports

## Audit criteria (pass/fail)
- [ ] Regulatory status determined (GDPR/NIS2/DORA/PCI).
- [ ] RACI + procedures documented.
- [ ] Evidence collected and accessible (links).
