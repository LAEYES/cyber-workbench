# STD — D08 — Registre d’Obligations (RGPD / NIS2 / DORA) — license-safe

**Organisation :** ACME  
**Version :** 0.2 (durci)  
**Date :** 2026-02-03

## 1. Objectif
Maintenir un registre unique, auditable, des obligations cyber légales/réglementaires et de leurs preuves.

## 2. Champs obligatoires (par obligation)
- Référentiel : RGPD / NIS2 / DORA / Autre
- Statut applicable (Oui/Non) + justification
- Périmètre (systèmes/processus/données)
- Délais & triggers (ex: notification incident)
- Owner (métier / IT / sécurité / juridique)
- Procédure associée
- Preuves attendues (liens)
- Date de dernière revue

## 3. Cadence
- Baseline : revue **trimestrielle**.
- Régulé : revue **mensuelle** sur périmètre critique.

## 4. Critères d’audit (pass/fail)
- [ ] Registre existe et est à jour.
- [ ] Owners et preuves renseignés.
- [ ] Revues réalisées dans les délais.

*Standard modèle : à compléter selon le statut juridique réel de ACME.*
